# Component Style Guide

## Navbar

- Use a sticky or fixed top navigation.
- Apply a translucent dark background with a subtle bottom border.
- Place the Vesperus wordmark on the left.
- Place navigation links on the right on desktop.
- Use a compact mobile menu on smaller screens.
- Include a restrained Contact CTA.
- Keep hover and focus states visible but subtle.

## Hero

- Use the strongest visual hierarchy on the page.
- Show the Vesperus name and tagline clearly in the first viewport.
- Use a dark premium background with restrained technical depth.
- Pair the headline with concise supporting copy.
- Include a primary CTA for NOVA.
- Use abstract technical visuals only if they support orchestration, validation, or telemetry concepts.

## CTA Buttons

- Primary CTAs use cyan or teal fill with dark text.
- Secondary CTAs use graphite or transparent surfaces with subtle accent borders.
- Use `999px` radius.
- Include clear hover, active, and keyboard focus states.
- Avoid oversized, playful, or flashy button styling.

## Proof Bar

- Place near the top of the homepage after the hero.
- Use compact labels or pills.
- Highlight validated orchestration, safety-first telemetry, Aivancity recognition, and decentralized healthcare infrastructure vision.
- Keep the layout horizontal on desktop and wrapped or stacked on mobile.

## Cards

- Use dark elevated surfaces.
- Use thin subtle borders.
- Use `24px` radius.
- Keep shadows soft and secondary to borders.
- Avoid nested cards.
- Use accent lines, icons, or small labels to create hierarchy.

## Section Headers

- Use a small technical eyebrow label.
- Use a clear heading with restrained language.
- Add one concise support paragraph when useful.
- Prefer left alignment for technical sections.
- Center alignment is allowed only for summary or transition sections.

## Timeline

- Use restrained status markers.
- Keep claims conservative.
- Recommended states include Architecture Defined, Gateway Integrated, Telemetry Validated, Safety Boundaries Documented, and Future R&D.
- Use a horizontal or split layout on desktop.
- Use a vertical layout on tablet and mobile.

## Recognition Card

- Present the Aivancity feature as external credibility.
- Include the article title and source.
- Use a clear external-link CTA.
- Use publication logo only if available and permitted.
- Avoid inflating the recognition into clinical validation.

## Founder Card

- Present Vishal Nelaturi clearly as founder of Vesperus.
- Include MSc Data Engineering at Aivancity Paris-Cachan.
- Include builder of NOVA and M.E.D.I.S.
- Include LinkedIn CTA.
- Use a real portrait if available.
- Avoid generic avatar illustrations.

## Footer

- Keep the footer minimal and structured.
- Include Vesperus, domain reference, navigation links, and relevant external links.
- Use muted text and subtle borders.
- Avoid oversized footer visuals or extra marketing claims.
